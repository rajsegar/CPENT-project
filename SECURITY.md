# Security and Responsible Use

This repository is for authorized learning and defensive validation.

## Safe-use boundary

Only test:

- Systems you own and have isolated for practice.
- Training platforms whose rules allow the activity.
- Client or employer systems covered by explicit written authorization and
  rules of engagement.

Do not test a public system merely because it appears vulnerable. Stop if a
target, technique, time window, or data type is outside the agreed scope.

## Protect study evidence

- Replace credentials and tokens with REDACTED.
- Avoid committing packet captures, browser archives, private keys, or raw
  customer exports.
- Use synthetic names and lab-only addresses in examples.
- Store sensitive evidence in an approved encrypted location, not in Git.

## Reporting a problem in this repository

If a document exposes a real secret or target, remove the sensitive value from
your working copy and notify the repository owner privately. Do not repeat the
secret in an issue, pull request, or commit message.
