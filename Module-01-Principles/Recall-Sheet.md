# Module 01 One-Page Recall Sheet

## Main idea

Penetration testing is an authorized simulated attack used to find, prove, document, and help fix security risks.

## Core definition

```text
Penetration testing = authorized simulated attack to prove real security impact.
```

## Vulnerability scanning vs penetration testing

```text
Vulnerability scanning:
Finds possible known weaknesses using automated tools.

Penetration testing:
Uses human analysis to validate exploitability, prove impact, collect evidence, and recommend remediation.
```

## Authorization

```text
No written authorization = no testing.
```

Authorization protects the tester, the client, and the systems being tested. It should define scope, testing windows, allowed actions, prohibited actions, evidence handling, emergency contacts, cleanup, and reporting responsibilities.

## Three main phases

```text
1. Pre-Attack
2. Attack
3. Post-Attack
```

## Pre-Attack phase

The tester prepares before exploitation.

Includes:

```text
Planning
Scoping
Rules of engagement
Reconnaissance
Scanning
Enumeration
Target understanding
Evidence planning
```

Simple memory:

```text
Pre-Attack = prepare and understand the target.
```

## Attack phase

The tester safely validates weaknesses inside the approved rules.

Includes:

```text
Exploitation
Gaining access
Privilege escalation
Lateral movement
Controlled post-exploitation
```

Simple memory:

```text
Attack = safely prove exploitability and impact.
```

## Post-Attack phase

The tester documents and restores.

Includes:

```text
Evidence collection
Cleanup
Reporting
Business impact explanation
Remediation advice
Retesting if required
```

Simple memory:

```text
Post-Attack = clean up, report, and help fix the issue.
```

## EC-Council LPT methodology

```text
1. Planning and Scoping
2. Reconnaissance and Information Gathering
3. Vulnerability Assessment
4. Gaining Access
5. Privilege Escalation
6. Lateral Movement
7. Maintaining Access
8. Data Exfiltration
9. Clearing Tracks
10. Documentation and Reporting
11. Remediation Testing
```

Simple memory:

```text
LPT methodology = EC-Council's structured process for a professional penetration test.
```

## MITRE ATT&CK

MITRE ATT&CK is a knowledge base of real-world adversary tactics, techniques, and procedures.

Simple memory:

```text
MITRE ATT&CK = map of real attacker behaviour.
```

## Compliance-driven penetration testing

Compliance-driven testing is performed to meet legal, regulatory, contractual, or industry requirements.

Examples:

```text
PCI DSS = payment card data
GDPR = personal data
HIPAA = healthcare data
SOC 2 = service organization controls
ISO/IEC 27001 = information security management system
```

## AI-driven penetration testing

AI can help with analysis, scripting, summarizing results, organizing evidence, and improving reports.

Important rule:

```text
AI assists the tester, but the human tester must verify everything.
```

## Final memory sentence

A professional penetration test starts with permission, scope, rules, safe testing, evidence collection, reporting, cleanup, and remediation.
