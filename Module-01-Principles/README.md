# Module 01: Introduction to Penetration Testing and Methodologies

## What this module teaches

This module explains what a penetration test is, why permission and scope come
first, and how a repeatable methodology turns technical activity into useful
business evidence.

## Learning checklist

- [ ] I can distinguish a vulnerability scan from a penetration test.
- [ ] I can explain authorization, scope, and rules of engagement.
- [ ] I can name the main testing phases.
- [ ] I can explain why evidence, cleanup, and reporting matter.
- [ ] I can connect technical findings to business impact.

## Beginner reminder

A good tester is not measured by how many tools they run. A good tester stays
inside the agreed boundaries, gathers reliable evidence, avoids unnecessary
harm, and explains what should happen next.
