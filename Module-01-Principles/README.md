## What this module teaches

This module explains what a penetration test is, why permission and scope come
first, and how a repeatable methodology turns technical activity into useful
business evidence.

## Beginner reminder

A good tester is not measured by how many tools they run. A good tester stays
inside the agreed boundaries, gathers reliable evidence, avoids unnecessary
harm, and explains what should happen next.
